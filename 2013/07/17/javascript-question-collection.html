情节

1.运行下面的代码，输出的结果是？ 



var a = 10;

sayHi();

function sayHi(){

    var a = a + 10;

    alert(a);

    return a;
}

alert(a);

alert(sayHi()+10);

答案是NaN, 10, NaN, NaN

分析：第一次的NaN，是因为var a在函数内部声明，js解析器遇到会提前解析，并把a的值设为undefined，此时的a已经在内部的作用域里找到了，就不用沿着作用域链向上找了，所以全局a不会影响函数内部的a，很自然的结果应该为：

第一次：undefined+10 //输出结果：NaN

第二次：因为局部的a在函数执行完销毁，在执行的时候也没对a造成影响，所以全局a保持其初值10 //输出结果：10
